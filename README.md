## 1. Общее описание проекта
 - это образовательная веб-платформа, которая объединяет:
-  интерактивные уроки по разным предметам в игровом формате (наподобие Duolingo);
-  библиотеку кратких пересказов книг и фильмов с интересными фактами;
-  телеграм-бота для быстрого поиска информации внутри платформы.

Проект создаётся в учебных целях, без использования личного сервера.

## 2. Цель
Сделать удобный сервис, где пользователь может:
- изучать школьные/университетские предметы в формате микроуроков;
- быстро находить и читать краткие содержания книг и фильмов;
- получать ту же информацию через телеграм-бота по запросу.

## 4. Архитектура (как всё работает без сервера)
- **База данных и API:** Supabase (бесплатная облачная БД с автоматическим REST API).
- **Сайт:** статические файлы HTML/CSS/JS, размещены на GitHub Pages. Сайт напрямую подключается к Supabase.
- **Телеграм-бот:** Python-скрипт (aiogram), который обращается к той же базе Supabase. Размещён на бесплатном хостинге PythonAnywhere / Render.


## 5. Функциональные требования

### 5.1. Сайт
1. **Регистрация и вход** — по email или через соцсети (используется Supabase Auth).
2. **Главная страница** — список доступных предметов и переход в раздел «Пересказы».
3. **Уроки:**
   - каждый предмет содержит дерево уроков/навыков;
   - урок состоит из коротких заданий (выбор ответа, ввод текста);
   - система начисляет баллы, открывает следующие уроки при успешном прохождении.
4. **Библиотека пересказов:**
   - список книг и фильмов с поиском по названию;
   - страница произведения: краткое содержание (текст) и блок «Интересные моменты»;
   - возможность сохранить в избранное.
5. **Личный кабинет:** прогресс обучения, статистика, настройки.
6. **Глобальный поиск** по урокам и пересказам.

### 5.2. Телеграм-бот
1. Команда `/start` — приветствие и подсказки.
2. Поиск по текстовому запросу: бот ищет в таблицах уроков и пересказов через Supabase API.
3. Ответ содержит:
   - название найденного материала;
   - краткий фрагмент (первые 500 символов);
   - ссылку на полную страницу сайта.
4. Команда `/progress` — показывает прогресс пользователя (если аккаунт Telegram привязан к аккаунту на сайте).

## 6. Нефункциональные требования
- Сайт должен корректно отображаться на мобильных устройствах (адаптивная вёрстка).
- Время загрузки любой страницы — не более 2 секунд.
- Безопасность: все пароли хешируются, данные передаются по HTTPS.
- Бот должен быть доступен 24/7 (реализуется средствами хостинга).
